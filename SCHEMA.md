# Candor Database Schema

이 문서는 현재 서비스에서 사용하는 핵심 테이블 중 공약(pledge) 도메인의 표준 구조를 정리합니다.

## Tables

### Core Entities
- elections
- candidates
- candidate_elections

### Pledges
- pledges
- pledge_nodes
- pledge_node_progress

### Related Tables
- pledge_node_progress_sources
- pledge_node_sources
- pledge_votes
- sources
- user_profiles
- contributor_stats
- reports
- terms

---

## elections

Stores election master records used by candidate/election mapping.

| column | type | default | description |
|------|------|------|------|
| id | int8 (PK) | generated by DB | Unique identifier |
| election_type | text | NULL | Election type (currently fixed to `대통령` in app) |
| title | int2 | NULL | Election round number |
| election_date | date | NULL | Official election date |
| created_at | timestamptz | now() | Record creation time |
| created_by | uuid | NULL | User who created the record |
| updated_at | timestamptz | NULL | Last update time |
| updated_by | uuid | NULL | User who last updated the record |

### Primary Key
- `id`

### Relationships
- `created_by` -> `user_profiles.user_id`
- `updated_by` -> `user_profiles.user_id`

### Notes
- `title` now stores only the election round number as `int2` (range: `1` to `32767`).
- UI display label should be rendered as `제{title}대 대통령 선거`.

## pledges

Stores top-level pledge information made by candidates in a specific election.

| column | type | default | description |
|------|------|------|------|
| id | uuid (PK) | gen_random_uuid() | Unique identifier |
| title | text | NULL | Title of the pledge |
| raw_text | text | NULL | Original pledge text |
| category | text | NULL | Policy category |
| timeline_text | text | NULL | Original timeline or implementation schedule text |
| finance_text | text | NULL | Original funding or financing text |
| parse_type | text | NULL | Parsing format used for this pledge (`type1`, `type2`, `type3`) |
| structure_version | int2 | 1 | Version of parsing / structure logic |
| fulfillment_rate | int4 | 0 | Overall fulfillment rate (%) |
| status | text | 'active' | Pledge status |
| candidate_election_id | int8 | NULL | Reference to candidate election |
| sort_order | int2 | NULL | Display order |
| created_at | timestamptz | now() | Record creation time |
| created_by | uuid | NULL | User who created the record |
| updated_at | timestamptz | NULL | Last update time |
| updated_by | uuid | NULL | User who last updated the record |

### Primary Key
- `id`

### Relationships
- `candidate_election_id` -> `candidate_elections.id`
- `created_by` -> `user_profiles.user_id`
- `updated_by` -> `user_profiles.user_id`

### Notes
- `category` stores top-level policy areas such as economy, welfare, education, diplomacy, politics, or administration.
- `timeline_text` and `finance_text` store original text from sections like `이행기간`, `재원조달방안`.
- `parse_type` records which parser rule set was used.
- `fulfillment_rate` is the summarized overall rate for the entire pledge.

## pledge_nodes

Stores the hierarchical structure of pledge content.
Each pledge is organized as a tree-like structure: Goal -> Strategy -> Action.

| column | type | default | description |
|------|------|------|------|
| id | uuid (PK) | gen_random_uuid() | Unique identifier |
| pledge_id | uuid | NULL | Reference to pledge |
| parent_id | uuid | NULL | Parent node (self-referencing tree structure) |
| node_type | text | NULL | Node type (`goal`, `strategy`, `action`) |
| level | int4 | NULL | Node level (`1=goal`, `2=strategy`, `3=action`) |
| name | text | NULL | Display title or summary of the node |
| content | text | NULL | Full original text or extended description |
| sort_order | int4 | 0 | Order of nodes among siblings |
| created_at | timestamptz | now() | Record creation time |
| created_by | uuid | NULL | User who created the record |
| updated_at | timestamptz | NULL | Last update time |
| updated_by | uuid | NULL | User who last updated the record |

### Primary Key
- `id`

### Relationships
- `pledge_id` -> `pledges.id`
- `parent_id` -> `pledge_nodes.id`
- `created_by` -> `user_profiles.user_id`
- `updated_by` -> `user_profiles.user_id`

### Notes
- `goal` nodes usually represent high-level purpose or intended outcome.
- `strategy` nodes represent implementation directions or grouped methods.
- `action` nodes represent concrete execution items under a strategy.
- `parent_id` is `NULL` for top-level nodes.

Recommended structure guideline:
- `goal` nodes: `parent_id = NULL`
- `strategy` nodes: `parent_id = NULL` or `goal` 하위로 구성 가능 (서비스 정책에 따라 선택)
- `action` nodes: `parent_id = strategy.id`

Additional guidance:
- `name` should store short display text.
- `content` can store full sentence, original line, or expanded description.
- `level` and `node_type` are both stored for easier parsing, validation, and UI rendering.

## pledge_node_progress

Stores evaluation records for a specific pledge node.

| column | type | default | description |
|------|------|------|------|
| id | uuid (PK) | gen_random_uuid() | Unique identifier |
| pledge_node_id | uuid | NULL | Reference to pledge node |
| progress_rate | numeric(5,2) | NULL | Progress percentage (0-100) |
| status | text | NULL | Progress status (`planned`, `in_progress`, `completed`, `failed`, `partially_completed`, `suspended`) |
| reason | text | NULL | Explanation of the evaluation |
| evaluator | text | NULL | Name of evaluator or evaluating organization |
| evaluation_date | date | NULL | Date of evaluation |
| created_at | timestamptz | now() | Record creation time |
| created_by | uuid | NULL | User who created the record |
| updated_at | timestamptz | NULL | Last update time |
| updated_by | uuid | NULL | User who last updated the record |

### Primary Key
- `id`

### Relationships
- `pledge_node_id` -> `pledge_nodes.id`
- `created_by` -> `user_profiles.user_id`
- `updated_by` -> `user_profiles.user_id`

### Notes
- 이 테이블은 단일 상태 테이블이 아니라 **평가 이력(history)** 테이블입니다.
- 하나의 `pledge_node`는 시간에 따라 여러 개의 progress 레코드를 가질 수 있습니다.
- 최신 레코드를 현재 상태로 간주할 수 있습니다.
- `progress_rate`는 `37.50` 같은 소수값을 허용합니다.

## Recommended Parsing and Storage Rules

### 1) Pledge-level storage (`pledges`)
Store:
- overall title
- original raw text
- category
- timeline text
- finance text
- parser type
- overall fulfillment rate

### 2) Node-level storage (`pledge_nodes`)
Store structured tree:
- goals
- strategies
- actions

### 3) Evaluation storage (`pledge_node_progress`)
Store evaluation history:
- progress rate
- evaluation reason
- evaluator
- evaluation date

---

## Recommended Parsing Interpretation

### Supported hierarchy
- `goal`
- `strategy`
- `action`

### Metadata sections
The following sections should usually be stored at pledge level (`pledges`), not in `pledge_nodes`:
- `이행기간`
- `재원조달방안`

### Supported parser formats
- `type1`: numbered strategy format (`①`, `②`, `③`)
- `type2`: bullet hierarchy (`□`, `○`, `-`)
- `type3`: simplified list-based format

- `goal`
- `strategy`
- `action`

### pledge_nodes.level

- `1`

### pledge_node_progress.status
Allowed values:
- `planned`
- `in_progress`
- `completed`
- `failed`
- `partially_completed`
- `suspended`

### pledge_node_progress.progress_rate
Recommended range:
- `0` to `100`

## Recommended Indexes

### pledges
- `(candidate_election_id, sort_order)`
- `(category)`
- `(status)`

### pledge_nodes
- `(pledge_id, level, sort_order)`
- `(pledge_id, node_type)`
- `(parent_id, sort_order)`

### pledge_node_progress
- `(pledge_node_id, evaluation_date DESC)`
- `(status)`
- `(created_at DESC)`

---

## Design Rationale

- `pledges` stores one pledge as a top-level unit.
- `pledge_nodes` stores structured content as a flexible tree.
- `pledge_node_progress` stores evaluation history separately, allowing repeated assessments.
- 이 구조는 대통령 공약뿐 아니라 향후 국회의원/기타 직책 공약으로도 확장 가능하며, 파서가 변경되어도 DB 스키마는 안정적으로 유지됩니다.
